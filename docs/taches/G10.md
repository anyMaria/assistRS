- Modèle recommandé : **Sonnet**.

## Objectif de ce bloc

Le planning devient le centre de l'app : on y gère ses vues comme dans Notion, on y
voit d'où vient chaque publication (l'idée d'origine), et l'app y propose les bons
horaires. Tout ce qui touchait aux vues dans Paramètres vit désormais ici.

## 1. Vues à la volée (Idées **et** Planifier)

Barre d'outils au-dessus des vues, sur `/idees` et `/planning` :

- Onglets des vues existantes (nom + icône du type) + bouton « + Nouvelle vue » qui crée
  la vue **en place** (petit popover : nom, type table/kanban/calendrier/galerie) —
  réutilise `createView`, plus besoin d'aller ailleurs.
- Sur chaque onglet, menu contextuel (icône `MoreHorizontal`) : renommer, dupliquer,
  supprimer (confirmation).
- Bouton « Filtres & tri » ouvrant un panneau (popover) qui édite la config de la vue
  active : tri, filtres, regroupement, propriétés affichées — réutilise
  `updateViewSettings` / `parseViewSettings` / `applyViewSettings` existants, avec
  application immédiate (submit du formulaire = revalidation de la page).
- Bouton « Couleurs » dans la même barre : gère les règles conditionnelles de l'entité
  courante (réutilise `createColorRule` / `deleteColorRule`).
- Filet de sécurité : s'il n'existe **aucune vue** pour une entité, en créer une par
  défaut automatiquement (table) au chargement au lieu d'afficher « Aucune vue
  configurée — va dans les paramètres ».

## 2. Pipeline idée → publication → Buffer, visible

La liaison existe déjà en base (`publications.ideaId`, action `planifierIdee`) mais
elle est invisible. La rendre explicite :

- Sur une carte/ligne d'**idée** : bouton principal « Planifier → » (accent) qui ouvre
  le formulaire de planification (date/heure + plateforme préremplie). Après création,
  passer l'idée en statut `en_production` automatiquement, et afficher sur l'idée un
  lien « Voir la publication » vers le planning.
- Sur une **publication** issue d'une idée : badge/lien « Depuis l'idée : {titre} »
  dans le panneau de détail, qui renvoie vers l'idée.
- Chemin d'étapes affiché dans le détail d'une publication (petit stepper visuel) :
  Idée → Planifiée → (visuel produit, checklist) → Envoyée à Buffer → Publiée → Stats.
  L'étape « Envoyée à Buffer » s'appuie sur `bufferPostId` (voir G11 §1 — si G11 n'est
  pas encore fait, afficher l'étape seulement si la colonne existe).
- Sur la page **Créer** : après génération, chaque idée retenue affiche deux actions :
  « Garder en idée » (comportement actuel) et « Planifier tout de suite ».
- Sur **S'inspirer** : chaque élément d'inspiration gagne une action « En faire une
  idée » (crée une idée préremplie : titre, thème, lien source dans `content`).

## 3. Horaires suggérés dans le formulaire (plus de module à part)

- Dans le formulaire de publication (création et planification d'idée), sous le champ
  date/heure : afficher 3 **suggestions cliquables** (`Tag` accent-soft) du type
  « mar. 11h — créneau conseillé » calculées pour la plateforme choisie :
  d'abord via `computeRealSlots` (analyse des stats réelles, si ≥ `MIN_PUBS_FOR_ANALYSIS`
  publications avec stats), sinon fallback sur les créneaux `time_slots` (grille Buffer).
  Cliquer remplit le champ date/heure (prochaine occurrence du créneau).
- Dans la vue **calendrier**, marquer discrètement les créneaux conseillés du mois
  (point accent dans la case du jour, tooltip « Instagram : 11h conseillé »).
- Supprimer l'ancien module interactif d'analyse d'horaires de l'ex-page Analyse
  (la partie `computeRealSlots` de la page) — c'est remplacé par ces suggestions.
