# Tâche G11 — Mesurer : statistiques visuelles (recharts) + synchro automatique via l'API Buffer

## Avant de commencer (obligatoire)

- G8 à G10 terminés et validés.
- Lis `src/lib/buffer.ts` (client GraphQL existant), `src/app/actions/buffer.ts`,
  `src/lib/kpi.ts`, `src/db/schema.ts` (`publications`, `stat_snapshots`),
  `src/app/api/cron/quotidien/route.ts`. Référence visuelle : maquette
  `pages/Mesurer.tsx` (recharts, `ChartBox`, adaptation clair/sombre) et
  `pages/Accueil.tsx`.
- Doc API : `https://developers.buffer.com/reference.html` — champs `Post.metrics`
  et query `aggregatedPostMetrics` (statut **Preview/expérimental** : isole tout
  dans `src/lib/buffer.ts` pour encaisser un changement de schéma sans toucher au reste).
- Modèle recommandé : **Sonnet**.

## Objectif de ce bloc

Remplacer la saisie manuelle de stats comme voie principale par une synchro Buffer,
et rendre les pages Mesurer + Accueil réellement visuelles (graphiques interactifs).

## 1. Relier les publications aux posts Buffer

- Migration Drizzle : colonne `buffer_post_id` (text, nullable) sur `publications`.
- `envoyerSurBuffer` reçoit déjà l'id du post créé (`result.data.id`) mais le jette :
  **le stocker** dans `buffer_post_id` et mémoriser aussi que l'envoi a eu lieu.
- Prévoir un rattachage manuel : dans le détail d'une publication publiée sans
  `buffer_post_id`, petit champ « ID de post Buffer » (optionnel, pour l'historique).

## 2. Synchro des métriques Buffer → `stat_snapshots`

- Dans `src/lib/buffer.ts`, ajouter `getPostMetrics(postIds)` : query GraphQL `post(s)`
  avec sélection `metrics` (types normalisés : impressions, reach, reactions,
  comments, shares, clicks, saves, follows, engagementRate + `metricsUpdatedAt`).
- Mapping vers `stat_snapshots` : impressions→impressions, reach→reach,
  reactions→likes, comments→comments, shares→shares, saves→saves, clicks→clicks,
  follows→followersGained. `conversions` reste manuel (saisie inchangée).
- Un module `src/lib/buffer-sync.ts` : pour chaque publication `publiee` avec
  `buffer_post_id`, créer un **nouveau relevé** si `metricsUpdatedAt` est plus récent
  que le dernier relevé de la publication (on garde l'historique multi-relevés, c'est
  le modèle existant). Logger dans `api_usage`.
- Déclenchement : 1) dans le cron quotidien existant ; 2) bouton « Synchroniser
  Buffer » (icône `RefreshCw`) en haut de Mesurer, avec retour visuel du nombre de
  relevés créés. Les métriques Buffer datent de ~24 h — l'afficher (« données Buffer
  du {date} »).
- La saisie manuelle de relevés reste disponible (fallback pour les publications
  hors Buffer) mais passe en secondaire (dans le détail d'une publication).

## 3. Page Mesurer visuelle

Installer `recharts`. Reprendre la structure de la maquette `Mesurer.tsx` :

- **Filtres persistants en haut** : marque (Tous + une par marque, pastille couleur),
  plateforme, période (30 j / 90 j / 12 mois).
- **Graphique principal** (courbe) : engagement % dans le temps, une courbe par
  plateforme (couleurs plateformes) — construit à partir des `stat_snapshots` agrégés
  par mois/semaine selon la période.
- **Graphique barres** : portée (ou abonnés gagnés) par mois, sélecteur de métrique.
- **Cartes KPI** au-dessus (engagement moyen, portée cumulée, abonnés gagnés,
  publications avec stats) — mêmes agrégats que `aggregate()` actuel.
- Utilise le pattern `ChartBox` de la maquette (ResizeObserver, pas de rendu à 0×0)
  et ses couleurs d'axes/tooltips adaptées au mode sombre.
- En dessous : la liste des publications avec leurs relevés (existant, restylé G8).

## 4. Accueil : vue d'ensemble graphique, sans rangée de CTA

- Remplacer les simples chiffres de `GlobalStats` par : les 4 cartes KPI compactes
  **avec sparkline** (mini-courbe 90 jours dans chaque carte) + un mini-graphique
  d'engagement global (courbe, 3 mois) cliquable vers `/mesurer`.
- **Supprimer la rangée de raccourcis CTA** en bas du tableau de bord (« ＋ Saisir des
  stats », « ✎ Trouver une idée »…) — la nav et le bouton « + » global suffisent.
- Garder : blocs À produire / À relancer / À relever, KPI par marque, objectifs.

## Critères d'acceptation

- [ ] Mesurer affiche courbe + barres + KPI, filtrables par marque/plateforme/période,

- [ ] L'Accueil n'a plus de rangée de CTA et montre des graphiques.
