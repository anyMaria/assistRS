# Tâche G12 — Retours clients (dans la marque) + pense-bête d'idées

## Avant de commencer (obligatoire)

- G8 à G11 terminés et validés (G12 peut suivre G9 si besoin, il est indépendant de G10/G11).
- Lis `src/db/schema.ts` (tables `brand_*`, `ideas`), `src/app/(app)/marques/[id]/page.tsx`
  (structure en onglets), `src/app/actions/brand.ts`, `src/lib/recall.ts`
  (la « mémoire de corrections » existante — à ne pas dupliquer).
- Modèle recommandé : **Sonnet**.

## Objectif de ce bloc

Deux zones de texte libres qui manquent au quotidien : les retours des clients sur les
publications livrées (rangés dans l'espace de marque), et un pense-bête personnel pour
les idées en vrac (rangé dans la bibliothèque d'idées).

## 1. Retours clients — nouvel onglet dans l'espace de marque

- Table `brand_client_notes` : `id`, `accountId` (FK cascade), `publicationId`
  (FK `set null`, **optionnel** — un retour peut être général), `content` (text),
  `createdAt`. Migration Drizzle.
- Onglet « Retours client » ajouté aux onglets existants de `/marques/[id]`
  (Contexte / Identité / Éditorial / Mémoire / **Retours client**) :
  - Formulaire simple en haut : textarea + sélecteur optionnel « À propos de la
    publication… » (liste des publications de la marque, plus récentes d'abord).
  - Liste antichronologique des retours ; si lié à une publication, badge cliquable
    vers son détail dans le planning. Suppression par bouton-icône + confirmation.
- Dans le détail d'une **publication** (planning), afficher les retours qui lui sont
  liés + raccourci « Ajouter un retour client ».
- **Distinction avec la « Mémoire de corrections »** (règles réutilisées par l'IA) :
  les retours clients sont du **verbatim archivé**, pas des règles. Ajouter sous la
  liste un bouton discret « Transformer en règle de mémoire » qui pré-remplit le
  formulaire de `brand_memory_rules` avec le texte du retour.

## 2. Pense-bête d'idées — notes rapides dans `/idees`

Décision d'Ana : ces notes vivent dans la base d'idées (pas un système à part), comme
étape amont d'une vraie idée.

- Table `idea_notes` : `id`, `accountId` (FK, **optionnel** — une note peut être
  générale), `content` (text), `createdAt`.
- Sur `/idees`, au-dessus des vues : bloc repliable « Pense-bête » (icône `StickyNote`)
  affichant les notes en petites cartes (fond `accent-soft` clair, style post-it sobre) :
  - Ajout ultra-rapide : un champ texte + Entrée, sans autre champ obligatoire.
  - Sur chaque note : bouton « → En faire une idée » (ouvre le formulaire d'idée
    prérempli avec le contenu, puis supprime la note à la création) et bouton-icône
    supprimer + confirmation.
- Ajouter « Note pense-bête » au menu du bouton « + » global (capture rapide) pour
  pouvoir noter une idée depuis n'importe quelle page.
- La recherche ⌘K inclut les notes (nouveau groupe « Pense-bête »).

## Critères d'acceptation

- [ ] Un retour client peut être saisi depuis la marque ET depuis une publication,
      avec ou sans publication liée ; visible aux deux endroits.
- [ ] « Transformer en règle de mémoire » crée bien une règle `brand_memory_rules`.
- [ ] Une note pense-bête se crée en < 5 secondes depuis n'importe quelle page
      (bouton « + » global) et se convertit en idée en un clic.
- [ ] Migrations propres (`npm run db:push` sans perte de données existantes).
- [ ] `npx tsc --noEmit` et `npm run build` passent.
