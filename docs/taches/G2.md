## Objectif de ce bloc

Amener l'IA (Google Gemini, palier gratuit) dans l'app : génération d'idées, rédaction
de légende assistée par le contexte de marque, mémoire de corrections qui s'auto-enrichit,
recall déterministe des publications qui ont bien marché, et déclinaison multi-formats.

## Ce qu'il faut construire

- `src/lib/gemini.ts` : client unique `@google/genai` (à installer), modèle configurable
  via `GEMINI_MODEL` (défaut `gemini-2.5-flash`) — voir le code exact en
  `GUIDELINE.md` §5.
- `src/lib/brand-context.ts` : `buildBrandContext(accountId)` → texte compact (< 600
  mots) assemblant contexte marque + ligne éditoriale + règles de mémoire actives.
  Écrit une seule fois, réutilisé par toutes les features IA ci-dessous.
- Génération d'idées sur `/creation` : marque + thème → 3-5 idées structurées (JSON,
  `GUIDELINE.md` §5.2) → cartes avec bouton « Retenir » → insertion dans `ideas`
  (source `ia`).
- Rédaction de légende : bouton « Rédiger la légende » sur une publication/idée →
  `POST /api/ia/legende` → éditeur inline pré-rempli, deux actions : **Appliquer**
  (enregistre dans `generations.editedOutput` + copie sur le brief de la publication) et
  **Mémoriser mes corrections** (`GUIDELINE.md` §5.4 : envoie généré + corrigé à l'IA,
  reçoit 1-3 règles courtes, affichées cochables, insérées dans `brand_memory_rules`
  après confirmation).
- Bouton optionnel « Améliorer mon texte » (jamais automatique) — prompt `GUIDELINE.md`
  §5.5.
- Recall (`src/lib/recall.ts`, **aucun appel IA**) : détection déterministe (mot
  commun > 3 lettres entre thème/pilier et titre/pilier d'anciennes publications,
  engagement du dernier relevé > médiane de la marque) → encart « Ça avait bien marché ».
- Déclinaison : bouton « Décliner vers… » → copie la publication vers une autre
  plateforme (statut planifiée, sans date) + affiche le rappel de format
  (`src/lib/formats.ts`, tableau `CONCEPTION.md` §4.4).
- Table `generations` alimentée à chaque appel IA (schéma déjà dans `CONCEPTION.md` §10).
